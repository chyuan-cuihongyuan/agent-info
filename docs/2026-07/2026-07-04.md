# 2026-07-04 AI 资讯

| 研究方向 | 大概讲了什么 | 文章链接 |
|---------|------------|---------|
| 多 Agent/推理 | 将多 Agent 潜在推理中的 KV-cache 合并建模为收敛复制状态，提出 CanonicalMerge 按内容排序使合并在任意输入排列下字节一致，在 Qwen3-1.7B/4B 上验证并匹配最优排序精度 | [Cache Merging as a Convergent Replicated State for Multi-Agent Latent Reasoning](https://arxiv.org/abs/2607.01308) |
| 训练方法 | HERMES 提出多粒度分层标签体系用于预训练数据混合，残差向量量化生成粗到细编码，在 1B 参数 25B token 预训练中发现粒度与混合规则的交互效应 | [HERMES: A Multi-Granularity Labeling Substrate for Pre-training Data Mixtures](https://arxiv.org/abs/2607.02266) |
| 人机协作 | 分析 53 篇人机协作论文，基于心理学分类法归纳出五种团队类型（AI 助手、临时依赖、强制依赖、配对均衡、群体均衡），质疑不同研究间结论的可迁移性 | [What Types of Human-AI Teams Exist?](https://arxiv.org/abs/2607.02198) |
| AI 安全/对齐 | 综述 AI 风险评估与管理方法论，梳理全球监管框架与技术到伦理社会层面的风险谱系，指出方法论差距与最佳实践 | [Overview of Risk Assessment and Management for Intelligent Systems under the AI Act and Beyond](https://arxiv.org/abs/2607.02197) |
| LLM/文化 | 从文学研究视角构建文化 AI 框架，将比较阅读、叙事学、批判理论与世界文学方法引入 LLM 文化素养构建，应对结构化单语主义 | [World Wide Models: Literary Tools for Cultural AI](https://arxiv.org/abs/2607.02369) |
| 评估基准 | 分析 650 篇使用 LLM-as-a-Judge 的论文，发现仅 33 篇关注低资源语言，揭示多语言场景下过度信任、评估不一致与单一裁判模型依赖问题 | [Challenges and Recommendations for LLMs-as-a-Judge in Multilingual Settings and Low-Resource Languages](https://arxiv.org/abs/2607.02235) |
| Agent 框架 | CausalSteward 多 Agent 分治-结合框架用于因果发现，融合 RAG 与条件独立性检验，人在回路保障高维因果建模的可信度 | [CausalSteward: An Agentic Divide-Conquer-Combine Copilot for Causal Discovery](https://arxiv.org/abs/2607.01936) |
| 多模态/评估基准 | AnyGroundBench 面向 5 个专业领域（动物/工业/体育/手术/公安）的视频时序定位基准，评估 15 个 SOTA VLM 发现零样本与 ICL 适配均存在时空推理缺陷 | [AnyGroundBench: A Specialized-Domain Benchmark for Video Grounding in Vision-Language Models](https://arxiv.org/abs/2607.02269) |
| 人机协作 | 用预测市场研究人机混合智能，发现协作绩效呈三模态分布，人力资本（视角转换、智识谦逊、好奇心）而非模型基准预测协作效果 | [Human Capital, Not Model Benchmarks, Predicts Hybrid Intelligence in Forecasting](https://arxiv.org/abs/2607.02467) |
| 训练方法/领域 LLM | FitOne 基于 Qwen3 的健身领域 LLM 后训练，三阶段流水线（持续预训练+SFT+RL），在 ACSM-EP/NSCA-CSCS 认证考试上提升 7-12% | [Enhancing Fitness Intelligence through Domain-Specific LLM Post-Training](https://arxiv.org/abs/2607.02118) |

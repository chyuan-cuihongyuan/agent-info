# 2026-06-29 AI 资讯

| 研究方向 | 大概讲了什么 | 文章链接 |
|---------|------------|---------|
| Agent 框架 | 提出基于语义相似度与质量变化的早停机制，替代固定迭代上限，在多跳 RAG 任务上节省 38% token 开销 | [Semantic Early-Stopping for Iterative LLM Agent Loops](https://arxiv.org/abs/2606.27009) |
| Agent 安全 | 形式化定义提示组合型 Agent 系统中模块间的"指令泄漏"干扰，揭示 Transformer 自注意力缺乏模块边界的隐患 | [Instruction Bleed: Cross-Module Interference in Prompt-Composed Agentic Systems](https://arxiv.org/abs/2606.26356) |
| Agent 训练 | 提出智能体与评估器共同进化的框架，通过控制效用演化实现自我改进型 Agent | [The Red Queen Gödel Machine: Co-Evolving Agents and Their Evaluators](https://arxiv.org/abs/2606.26294) |
| Agent 应用 | 用 LLM 驱动的流水线对 DAO 与企业 AI 协议进行治理对比分析 | [Agentic Analysis for Agentic Infrastructure](https://arxiv.org/abs/2606.26203) |
| Agent 安全 | 对 ERC-8004 去中心化 AI Agent 生态系统进行实证研究，评估去中心化 Agent 的可信度 | [Can Trustless Agents Be Trusted?](https://arxiv.org/abs/2606.26028) |
| Agent 框架 | 将 Agent 机制引入物理约束下的基础模型演化，实现模型在硬件约束下的自适应进化 | [Agentic evolution of physically constrained foundation models](https://arxiv.org/abs/2606.25532) |
| Agent 应用 | 用多 Agent LLM 架构在严肃游戏中对金融素养进行隐蔽评估 | [Agentic Knowledge Tracing: A Multi-Agent LLM Architecture](https://arxiv.org/abs/2606.25358) |
| LLM 评估 | 从多维度多环境评估前沿 LLM 的操纵能力，揭示操纵表现的任务依赖性 | [Manipulation Is Task-Dependent](https://arxiv.org/abs/2606.25899) |
| LLM 评估 | 通过模型蒸馏评估 LLM 在静力学问题上的求解能力，发现引入图表后多步推理准确率显著下降 | [Investigating LLM's Problem Solving Capability](https://arxiv.org/abs/2606.26103) |
| LLM 推理 | 研究十种语言特征对 LLM 动物福利推理的影响，发现断言性语言比中性描述更能引导立场 | [Assert, don't describe](https://arxiv.org/abs/2606.26104) |
